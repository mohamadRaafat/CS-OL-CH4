What happens when the computer is switched on?

  • Initiating programs (Bootstrap) are loaded into ROM A check is made that Hardware, Software & BIOS are functioning correctly If no errors, OS is loaded into the memory

Note:

  • Simple devices such as household items have embedded microprocessors rather than an Operating system as they usually carry out a single simple task

Interrupts & Buffers

Interrupt:

  • signal
  • sent from a device/software to the processor
  • requiring the attention of the processor

Types:

  • Hardware Interrupt: Printer out of paper
  • Software Interrupt: error in a running program
  • User Interrupt: user press some keys to interrupt the current process

Processor doing a task Interrupt received processor checks the priority of the interrupt If the priority is high enough, the status of the current task is saved using the interrupt handler Processor serves the interrupt Continue the task

Buffer:

  • Temporary storage area
  • used to compensate for the speed gap between the very fast processor and the slow peripherals

Von Neuman architechture

Stored Program Concept:
  • Program is stored on a secondary storage device
  • Program consists of a set of instruction
  • The instructions are moved to main memory (RAM)
  • Instructions are then moved to registers to be executed
  • Processor executes each instruction one after another
Busses Registers PC MAR MDR CIR ALU
Data Bus:
  • Transports Data
  • Bidirectional
Address Bus:
  • Transports an Address
  • Unidirectional
Control Bus:
  • Carries signals relating to control and coordination
  • Can be Unidirectional or Bidirectional
  • High speed storage area
  • Data is represented in a register to be processed
  • examples: PC, MAR, MDR, CIR, ALU, ACC

Contains address of the next instruction to be fetched

Contains the address of the memory location of the instruction

  • Stores data that is in use
  • This data is in the address held in MAR

Stores the instruction WHILE it's being decoded and executed

  • Carries out Calculations and Logical Operations
  • During Calculations, data is held in a register called Accumulator (ACC)

What is the role of the Control unit (CU)?

  • The control Unit reads an instruction from the memory (Fetching)
  • The instruction is then interpreted (Decoding)
  • Signals are sent (with the control bus) to other computer components telling them what to do (Executing)

Fetch-Decode-Execute cycle
  1. PC contains the address of the next instruction to be fetched
  2. This address is copied to the MAR (using the address bus)
  3. The contents of the memory location held in MAR is copied to the MDR (using data bus)
  4. The contents are then copied to the CIR to be processed
  5. The value in the PC is then incremented by 1 so that it now points to the next instruction to be fetched

    6. This is the "Fetch" part of the cycle

  6. The instructions in CIR are decoded and executed by sending signals (Using control bus) to other computer components telling them what to do

PC MAR MDR CIR PC = PC + 1 Decode & Execute
